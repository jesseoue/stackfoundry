# Auth Core Checklist

- [ ] module manifest validates
- [ ] account mode is selected: personal, team, or hybrid
- [ ] primary auth adapter is selected: `clerk-auth` or `auth-better-auth`
- [ ] public, auth, console, and webhook route groups are documented
- [ ] session shape is documented
- [ ] protected route strategy is documented
- [ ] sign-in, sign-up, sign-out, webhook sync, and session expiry checks are assigned to the chosen adapter
